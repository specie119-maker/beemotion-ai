BeeMotion 단체모드 공통 운영 규칙

이 규칙은 손으로 잡고 넣기 운동뿐 아니라
앞으로 모든 단체모드 프로그램에 공통 적용합니다.

절대 부분 기능만 보고 수정하지 말고,
아래 전체 흐름을 기준으로 단체모드를 다시 정리해 주세요.

=====================
1. 단체모드 기본 원칙
=====================

단체모드는 팀 대항 릴레이 방식입니다.

팀:
- 빨강팀
- 파랑팀
- 초록팀

각 팀은 동일한 조건으로 운동해야 합니다.

예:
운동시간 10초 선택 시
빨강팀도 10초
파랑팀도 10초
초록팀도 10초

속도, 난이도도 세 팀 모두 동일하게 적용합니다.

=====================
2. 경기 시작 전 설정
=====================

단체모드 설정 화면에서 먼저 선택:

- 참가 팀: 빨강 / 파랑 / 초록 모두 사용
- 팀별 인원: 2명 / 3명 / 4명
- 운동시간: 10초 / 30초 / 60초
- 속도: 보통 / 빠르게

중요:
단체모드는 한 팀만 선택하는 구조가 아니라
빨강팀 → 파랑팀 → 초록팀 순서로 모두 진행하는 구조입니다.

=====================
3. 진행 순서
=====================

예:
인원 2명, 시간 10초 선택

진행 순서:

빨강팀 1번 참가자 10초
↓
빨강팀 2번 참가자 10초
↓
파랑팀 1번 참가자 10초
↓
파랑팀 2번 참가자 10초
↓
초록팀 1번 참가자 10초
↓
초록팀 2번 참가자 10초
↓
최종 결과 화면

=====================
4. 다음 참가자 화면
=====================

각 참가자가 끝나면:

수고하셨습니다.
빨강팀 1번 참가자 운동 완료
성공 00개
다음 참가자 준비하세요.

[다음 참가자 시작하기]

버튼을 누르면 메뉴로 돌아가지 않고
바로 다음 순서 참가자가 시작됩니다.

=====================
5. 팀 점수 저장
=====================

각 참가자의 성공 개수를 팀별로 누적합니다.

예:

빨강팀:
1번 5개
2번 7개
총점 12개

파랑팀:
1번 6개
2번 5개
총점 11개

초록팀:
1번 8개
2번 4개
총점 12개

=====================
6. 최종 결과
=====================

세 팀의 모든 참가자가 끝난 뒤에만 최종 결과 화면을 보여줍니다.

결과 화면:

🔴 빨강팀 12개
🔵 파랑팀 11개
🟢 초록팀 12개

1등:
빨강팀 / 초록팀 공동 승리

또는 단독 1등이면:
축하합니다.
빨강팀 승리하셨습니다.

=====================
7. 절대 금지
=====================

- 한 팀만 끝나고 결과보기 표시 금지
- 두 팀만 끝나고 종료 금지
- 세 번째 팀 시작 버튼 오류 금지
- 팀 선택할 때 기존 점수 초기화 금지
- 다음 참가자 시작 시 흰 overlay 남김 금지
- 설정 화면으로 돌아가서 흐름 끊기 금지

=====================
8. 구현 방식
=====================

단체모드는 currentTeamIndex와 currentPlayerIndex로 순서를 관리합니다.

teams = ["red", "blue", "green"]

currentTeamIndex = 0
currentPlayerIndex = 1

한 참가자 종료 시:
현재 팀 점수에 score 추가

그 다음:
currentPlayerIndex 증가

만약 currentPlayerIndex > players:
currentPlayerIndex = 1
currentTeamIndex 증가

만약 currentTeamIndex >= teams.length:
최종 결과 화면 표시

그 외:
다음 참가자 시작 화면 표시

=====================
9. 완료 기준
=====================

10초 설정 시:
빨강팀 전원 10초
파랑팀 전원 10초
초록팀 전원 10초

모두 완료 후 결과표가 나와야 합니다.

이 규칙을 기준으로 단체모드를 다시 정리해 주세요.
